# Platform Ops Domain

## 1. Covered modules

- `dashboard`
- `operation-log`
- `notification-center`

## 2. Common business objects

- 指标卡片、趋势图、榜单、筛选维度、分组织口径
- 审计日志、操作日志、审批轨迹、脱敏字段、查询条件
- 消息模板、收件人、模板变量、发送记录、失败重试

## 3. High-risk test concerns

### 3.1 客户端

- 页面展示、筛选条件、时间选择、图表刷新、分页与详情抽屉
- 消息模板配置页、发送记录页、重试按钮、收件人展示
- 日志查询页、轨迹详情、筛选组合、导出入口

### 3.2 账服

- 统计接口聚合口径、时间边界、缓存刷新、权限过滤
- 日志写入时机、字段完整性、脱敏规则、查询性能
- 模板渲染、发送路由、失败回执、重试策略、重复发送控制

## 4. Common regression expansion rules

- 新增或修改业务流程 -> 补查 `operation-log` 与可能的 `notification-center`
- 修改统计口径 -> 补查指标来源模块与 `role-permission`
- 消息模板或路由变更 -> 补查触发源模块、失败重试和日志

## 5. Typical case ideas

### 运营看板

- 指标展示、筛选统计、时间边界、分组织口径、权限差异
- 客户端关注页面刷新、图表展示、筛选联动
- 账服关注统计查询、口径计算、缓存和数据一致性

### 操作日志

- 写日志、查日志、按模块筛选、按操作人筛选、审批轨迹回放
- 客户端关注查询页面、字段展示、脱敏展示
- 账服关注日志字段完整性、查询条件处理、异常补偿

### 消息中心

- 模板配置、发送、失败重试、收件人路由、发送记录
- 客户端关注配置页、记录页、按钮和状态展示
- 账服关注消息发送链路、渲染、回执、重试幂等、日志
