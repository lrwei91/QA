# Skill: Analyze Requirement

## 触发场景

- 用户只想评估需求质量
- 需要快速了解测试覆盖范围
- 需求评审阶段的早期分析
- 生成正式用例前的预分析

## 职责

1. **模块拆分** - 识别功能模块和子模块
2. **风险识别** - 标记高风险场景和关键校验点
3. **测试点清单** - 输出测试点列表（非完整用例）
4. **关联影响分析** - 识别受影响的关联模块

## 输出内容

### 1. 模块拆分

```
【主模块】模块名称
【子模块】子模块 1, 子模块 2, ...
```

### 2. 风险等级评估

| 等级 | 说明 |
|------|------|
| P0 | 核心主流程，阻断性风险 |
| P1 | 重要功能，影响用户体验 |
| P2 | 边界场景，低影响 |

### 3. 测试点清单

每个功能点列出需要测试的场景：

```
【功能点】功能名称
- [ ] 正向主流程
- [ ] 异常输入校验
- [ ] 边界值测试
- [ ] 状态流转测试
- [ ] 权限测试
```

### 4. 关联影响模块

列出可能受影响的关联模块：

```
【关联模块】
- 模块 A：原因说明
- 模块 B：原因说明
```

## 不输出的内容

- 完整的测试用例表格
- 详细的操作步骤
- 详细的预期结果
- 可直接导出的 Excel 格式

## 输出格式示例

```
## 需求分析报告

### 模块识别
- 主模块：用户中心
- 子模块：个人资料、头像上传

### 平台分布
- 账服：接口校验、数据落库
- 客户端：页面展示、交互反馈

### 风险等级
| 功能点 | 风险等级 | 说明 |
|--------|----------|------|
| 头像上传 | P0 | 涉及文件安全校验 |
| 昵称修改 | P1 | 涉及敏感词过滤 |

### 测试点清单

【头像上传】
- [ ] 正向：上传合规图片
- [ ] 异常：上传超大文件
- [ ] 异常：上传非图片格式
- [ ] 异常：上传恶意文件
- [ ] 边界：最小尺寸图片
- [ ] 边界：最大尺寸图片

【昵称修改】
- [ ] 正向：修改为合规昵称
- [ ] 异常：包含敏感词
- [ ] 边界：最小长度
- [ ] 边界：最大长度
- [ ] 异常：包含特殊字符

### 关联影响
- 账号系统：头像 URL 变更可能影响缓存
- 内容审核：昵称需要走敏感词过滤

### 建议
1. 优先覆盖 P0 级头像上传安全校验
2. 注意文件类型校验的边界情况
3. 关联检查缓存更新逻辑
```

## 分析维度

### 1. 功能性

- 核心功能是否明确
- 输入输出是否清晰
- 业务规则是否完整

### 2. 可靠性

- 异常处理是否考虑
- 状态流转是否闭环
- 错误提示是否明确

### 3. 安全性

- 权限校验是否到位
- 敏感数据是否保护
- 文件上传是否校验

### 4. 兼容性

- 多端展示是否一致
- 数据格式是否兼容

### 5. 可测试性

- 是否有明确的验证标准
- 是否有可获取的测试数据
- 是否有可模拟的场景

## 何时使用本 Skill

- 需求评审阶段，需要快速评估测试范围
- 需求变更时，评估影响面
- 生成正式用例前，先确认测试重点
- 资源有限时，优先覆盖高风险场景
