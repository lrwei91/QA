# Skill: Augment Test Cases

## 触发场景

- 用户提供已有测试用例（文件或内容）
- 需要补充遗漏的测试场景
- 需求有变更或新增功能点

## 职责

1. **分析已有用例覆盖** - 理解已有用例覆盖了哪些场景
2. **找缺口** - 识别边界、异常、权限、状态流转等遗漏点
3. **生成新增用例** - 输出去重后的补充用例
4. **追加到原文件** - 将新增用例追加到原 Excel 文件末尾，新增行标黄

## 核心约束

### 1. 不修改已有用例

- 不改写已有用例内容
- 不重排已有用例顺序
- 不删除已有用例

### 2. 只追加

- 只输出新增测试用例
- 新增用例追加到原文件末尾
- 序号从原最后序号 +1 开始连续递增

### 3. 标记新增内容

- 新增用例行使用黄色背景（#FFFF00）填充
- 索引条目的 `updated_at` 更新时间戳

## 工作流程

### 第 1 步：读取已有用例

用户提供已有用例时，首先读取：

```
已选择：[模块名称] - [用例标题]
文件路径：[文件路径]
已有用例数量：XX 条
```

### 第 2 步：分析覆盖缺口

基于以下维度分析是否有遗漏：

| 维度 | 检查内容 |
|------|------|
| 正向主流程 | 是否已覆盖核心业务路径 |
| 异常场景 | 是否已覆盖参数异常、操作异常 |
| 边界值 | 是否已覆盖上下限、临界点、空值 |
| 状态流转 | 是否已覆盖状态变更全流程 |
| 权限角色 | 是否已覆盖不同角色的权限差异 |
| 前后端联动 | 是否已拆分客户端和账服视角 |

### 第 3 步：生成新增用例

仅输出有真正新增的场景，如果没有真正新增，返回空结果，不要重复旧用例。

用例格式：

| 字段 | 说明 |
|------|------|
| 序号 | 从原最后序号 +1 开始连续递增 |
| 平台 | `客户端` 或 `账服` |
| 模块 | 与原有用例保持一致 |
| 功能点 | 具体验证目标 |
| 前置条件（测试点） | 执行前必须满足的状态 |
| 操作步骤 | 可执行的操作序列 |
| 预期结果 | 明确可验证的结果 |
| 测试结果 | 留空 |
| 备注 | 用例类型标记，如 `【异常测试】【P1】` |

### 第 4 步：追加到 Excel

使用脚本将新增用例追加到原文件：

```bash
python3 engine/scripts/xlsx_append_and_highlight.py \
    existing.xlsx new_rows.json output.xlsx --highlight
```

### 第 5 步：更新索引

使用 `upsert_testcase_index.py` 更新索引，复用原条目 `id`，仅更新 `updated_at`。

## 输出格式

```
补充用例已完成：

新增用例数量：XX 条
- 账服：XX 条
- 客户端：XX 条

已追加到原文件：[文件路径]
新增行已标黄标识
索引已更新：outputs/testcase-index.json
```

## 避重规则

- 使用等价类判断是否已有同质化用例
- 对仅文案不同、路径相同、校验逻辑相同的场景，不重复生成
- 优先补齐遗漏的功能、异常、边界、状态和权限场景
