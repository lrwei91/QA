# 输出模板

## 1. 默认列结构

用户未指定模板时，严格按以下列顺序输出：

1. 序号
2. 平台
3. 模块
4. 功能点
5. 前置条件（测试点）
6. 操作步骤
7. 预期结果
8. 测试结果
9. 备注

## 2. 已确认的工作簿画像

当前已确认的 Excel 模板画像来自 `任务系统前后端优化.xlsx`，按该结构更新导出约定：

- 工作表名称：`测试用例`
- 冻结窗格：第 7 行以下冻结，数据区从 `A8` 开始滚动
- 固定区：第 `1` 至 `7` 行
- 数据区：第 `8` 行起
- 标准数据列：`A:I`
- `J` 列及右侧绘图 / 图片区域不属于标准数据列，写入时不得覆盖、删除或整体右移

如果用户提供的是同结构模板，优先沿用以下规则；如果用户提供的是不同模板，仍以用户模板实际结构为准。

## 3. 固定区契约

### 3.1 标题区

- `A1:F1` 为合并标题区，写文档标题
- 当前模板标题示例为需求名称

### 3.2 文档信息区

以下字段位于固定区，若用户未明确提供内容，默认保留为空，不自动推断填写：

- `A2` `测试平台`
- `E2` `测试日期`
- `A3` `系统&版本`
- `E3` `最后更新`
- `A4` `文档编写人`
- `E4` `文档测试人`
- `A5` `策划负责人`
- `E5` `审阅人员`
- `A6` `参考档`

对应结构要求：

- `B2:D2` 合并，填写 `测试平台`
- `B3:D3` 合并，填写 `系统&版本`
- `B4:D4` 合并，填写 `文档编写人`
- `B5:D5` 合并，填写 `策划负责人`
- `B6:F6` 合并，填写 `参考档`

### 3.3 统计区

模板右上角存在统计区，位于 `G1:H4`，必须保留：

- `G1` `测试总数量`
- `G2` `Pass`
- `G3` `Fail`
- `G4` `N/A`

统计区规则：

- `H1` 为总数公式，应等于 `H2+H3+H4`
- `H2` 为 `Pass` 数量统计
- `H3` 为 `Fail` 数量统计
- `H4` 为 `N/A` 数量统计
- 统计公式必须覆盖当前真实数据区，即从 `H8` 统计到最后一条数据行
- 若新导出的测试用例 `测试结果` 默认留空，则统计值允许暂时为 `0`

### 3.4 表头行

第 `7` 行为固定表头行，列顺序如下：

1. `A7` `序号`
2. `B7` `平台`
3. `C7` `模块`
4. `D7` `功能点`
5. `E7` `前置条件（测试点）`
6. `F7` `操作步骤`
7. `G7` `预期结果`
8. `H7` `测试结果`
9. `I7` `备注`

不得变更表头文案、顺序或行号。

## 4. 数据区写入规则

### 4.1 基础写法

- 数据从第 `8` 行开始连续写入
- `序号` 使用连续数字 `1, 2, 3...`
- `平台` 只能填写 `账服` 或 `客户端`
- `模块` 写功能所属模块，不要过于笼统
- `功能点` 写被验证的具体点，不要只写“功能测试”
- `前置条件（测试点）`、`操作步骤`、`预期结果` 必须可执行、可验证
- 多行内容使用 `1、2、3、` 的编号格式分行书写
- `测试结果` 默认保持空值，除非用户明确提供执行结果
- `备注` 至少标记用例类型，推荐同时写优先级，例如 `【功能测试】【P0】`

### 4.2 多行字段

以下列默认视为多行富文本列：

- `E` 前置条件（测试点）
- `F` 操作步骤
- `G` 预期结果
- `I` 备注

导出时要求：

- 写入真实换行符，不要把多行压成单行
- 必须开启单元格自动换行
- 不要把行高压缩为单行高度

**分隔符自动转换规则：**

生成用例时，若使用分号分隔步骤（如 `1、xxx；2、xxx；3、xxx`），导出 Excel 时会自动将分号转换为换行符：

- 全角分号 `；` (U+FF1B) → `\n`
- 半角分号 `;` (U+003B) → `\n`

转换范围：`前置条件（测试点）`、`操作步骤`、`预期结果`（当包含编号格式时）。

这确保每个编号步骤在 Excel 中单独显示一行，配合自动换行功能完整展示。

### 4.3 合并单元格策略

模板样例数据中存在分组合并，例如平台列纵向合并。生成新数据时遵循以下规则：

- 默认逐行写入，不强制新增合并单元格
- 若用户明确希望沿用平台分组展示，可仅在 `B` 列对同平台的连续行做纵向合并
- 除固定区外，`E/F/G/I` 等业务列默认不新增合并，除非用户明确要求且确有业务依据

### 4.4 样例数据处理

当前模板第 `8` 行起已有样例测试用例，属于可替换数据区，不属于固定区。

当用户要求基于该模板导出新用例时：

- 保留第 `1` 至 `7` 行固定区不变
- 将第 `8` 行及以后视为数据区
- 可替换现有样例数据，或在清空样例后重新写入新数据
- 不得把样例文案误当作固定模板内容保留下来

## 5. Excel 导出契约

若用户要求导出 Excel，且使用该模板或同结构模板：

- 保持工作表 `测试用例`
- 保持冻结窗格在 `A8`
- 保持标题区、文档信息区、统计区、表头行位置不变
- 保持现有合并区域不被破坏，尤其是 `A1:F1`、`B2:D2`、`B3:D3`、`B4:D4`、`B5:D5`、`B6:F6`
- 统计区公式应随最终数据区更新到最后一条记录
- 标准测试用例数据只写入 `A:I`
- `J` 列及右侧图片 / 绘图对象区域保持原样，不覆盖、不删除、不移动
- 若用户未要求嵌入截图或示意图，不要把右侧图片区当作测试用例数据列

## 6. 对话中的推荐输出结构

### 6.1 默认对话输出

优先使用 markdown 表格输出结构化测试用例；当单元格内容较长时，保留换行或使用 `<br>` 保持可读性。

### 6.2 分组规则

当存在关联回归用例时，按下面顺序组织：

1. `需求直接用例`
2. `关联补充用例`
3. `索引增量建议`（如有）
4. `模块文档增量建议`（如有）

### 6.3 增量补充场景

如果用户提供旧用例并要求补充：

- 只输出新增行
- 不重复旧用例
- 如无新增，返回空结果
