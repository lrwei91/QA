# Axure RP 导出指南

本指南说明如何从 Axure RP 导出 HTML 文件，以便 QA 测试用例生成器可以提取 UI 结构、元件信息和交互说明。

---

## 快速开始

### 步骤概览

1. 在 Axure 中完成原型设计
2. 配置导出选项（关键步骤）
3. 生成 HTML 文件
4. 验证导出内容
5. 提供给 `/qa` 命令使用

---

## 推荐导出设置

### 1. 打开发布菜单

```
发布 → 生成 HTML 文件...
```

### 2. 配置选项设置

在弹出窗口中，确保勾选以下选项：

```
✓ 生成原型说明
✓ 包含元件注释
✓ 包含交互说明
✓ 包含母版说明
✓ 按页面生成（推荐）
```

**关键选项说明：**

| 选项 | 作用 | 是否必需 |
|------|------|----------|
| 生成原型说明 | 生成导航和页面信息 | 必需 |
| 包含元件注释 | 提取元件的功能说明和约束条件 | 必需 |
| 包含交互说明 | 提取点击、跳转等交互行为 | 推荐 |
| 包含母版说明 | 提取全局组件（如导航栏、页脚） | 推荐 |
| 按页面生成 | 每个页面生成独立 HTML 文件 | 推荐 |

### 3. 选择输出目录

```
输出目录：testcases/axure_export/
```

建议将导出目录放在项目的 `testcases/axure_export/` 目录下，便于管理。

## 元件注释规范

### 如何添加注释

1. 选中元件
2. 右键 → 添加注释
3. 填写注释内容

### 注释内容建议

```
【功能说明】
说明元件的用途和功能

【输入约束】
- 格式要求（如：手机号 11 位数字）
- 长度限制（如：密码 6-20 位）
- 字符类型（如：仅支持字母和数字）

【交互效果】
- 点击后的行为
- 状态变化（如：Disabled → Active）
- 数据提交/刷新

【边界条件】
- 空值处理
- 错误状态
- 加载状态
```

### 注释示例

```
【功能说明】
用户登录按钮，点击后提交登录表单

【输入约束】
- 手机号和验证码填写完成后才能点击
- 未填写时按钮为 Disabled 状态

【交互效果】
- 点击后显示加载动画
- 登录成功跳转到首页
- 登录失败显示错误提示

【边界条件】
- 连续失败 5 次后锁定 30 分钟
- 网络超时时显示重试按钮
```

## 交互说明规范

### 如何设置交互

1. 选中元件
2. 在交互面板中新建交互
3. 配置触发事件和目标动作

### 常用交互类型

| 交互类型 | 触发事件 | 目标动作 |
|----------|----------|----------|
| 按钮点击 | onClick | 页面跳转 / 提交表单 / 显示弹窗 |
| 输入变更 | onChange | 实时校验 / 显示提示 |
| 鼠标悬停 | onMouseOver | 显示 Tooltip / 状态变化 |
| 页面加载 | onPageLoad | 加载数据 / 显示默认状态 |

### 交互标注建议

在交互说明中添加详细描述：

```
【触发条件】
用户点击登录按钮

【前置校验】
- 手机号格式正确
- 验证码已填写

【目标动作】
跳转到首页

【动画效果】
淡入淡出，300ms

【数据传递】
将用户 token 传递给首页
```

导出后的目录结构如下：

```
testcases/axure_export/
├── index.html              # 原型首页（导航）
├── login.html              # 登录页
├── home.html               # 首页
├── profile.html            # 个人中心
└── resources/              # 资源文件（脚本、样式等）
```

## 使用方式

### 方式 1：在 `/qa` 命令中提供目录

```
/qa 生成测试用例
  ↓
选择用例类型
  ↓
提供 Axure 导出 HTML 目录：testcases/axure_export/
```

### 方式 2：使用解析脚本

```bash
# 解析单个文件
python3 test-case-generator/scripts/parse_axure_html.py \
    testcases/axure_export/login.html

# 解析整个目录
python3 test-case-generator/scripts/parse_axure_html.py \
    testcases/axure_export/

# 递归解析（包括子目录）
python3 test-case-generator/scripts/parse_axure_html.py \
    testcases/axure_export/ --recursive

# 输出到 JSON 文件
python3 test-case-generator/scripts/parse_axure_html.py \
    testcases/axure_export/ \
    --output /tmp/axure_data.json
```

## 常见问题排查

### 问题 1：解析后无数据

**可能原因：**
- Axure 导出时未勾选"包含元件注释"
- HTML 文件不完整
- 导出格式不兼容

**解决方案：**
1. 在 Axure 中重新导出，确保勾选所有必需选项
2. 提供完整的导出目录（而非单个文件）
3. 尝试提供 Axure .rp 源文件另行处理

### 问题 2：元件信息缺失

**可能原因：**
- 元件未添加注释
- 元件类型为容器类（如 Frame、Group）

**解决方案：**
1. 为关键元件添加详细注释
2. 为交互元素（按钮、输入框等）单独标注

### 问题 3：交互说明未识别

**可能原因：**
- 交互使用自定义脚本实现
- 交互效果未使用 Axure 标准交互功能

**解决方案：**
1. 使用 Axure 标准交互功能重建交互
2. 在元件注释中手动描述交互行为

## 最佳实践

### 1. 使用母版管理全局组件

```
✓ 推荐：将导航栏、页脚做成母版
  - 修改一处，全局生效
  - 解析时可以识别全局组件

✗ 不推荐：每个页面复制粘贴导航栏
  - 维护成本高
  - 可能被识别为重复元件
```

### 2. 为每个页面单独导出

```
✓ 推荐：按页面生成 HTML 文件
  - 便于单独处理和引用
  - 文件更小，解析更快

✗ 不推荐：所有页面合并成一个 HTML
  - 文件过大
  - 难以定位具体页面
```

### 3. 使用有意义的命名

```
✓ 推荐：登录按钮、手机号输入框、记录表格
✗ 不推荐：矩形 1、输入框 123、未命名
```

### 4. 保持注释更新

```
✓ 推荐：需求变更后同步更新注释
✗ 不推荐：注释与实际设计不一致
```

## 与其他工具集成

### 结合 Figma 设计稿

```
优先级：
1. 需求文档（PRD/Word） - 主需求来源
2. Axure HTML - UI 结构和交互
3. Figma 设计稿 - 视觉细节和状态

使用方式：
/qa 生成测试用例
  → 提供 Axure HTML 目录
  → 提供 Figma 链接（可选，补充视觉细节）
```

### 结合需求文档

```
推荐流程：
1. 提供 PRD/需求文档文本
2. 提供 Axure HTML 目录
3. （可选）提供 Figma 链接

合并策略：
- 模块识别：以需求文档为准
- UI 元素：Axure + Figma 合并
- 约束条件：Axure 注释优先（最详细）
- 视觉状态：Figma 优先
```
