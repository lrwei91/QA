# 充值提现领域

## 1. 覆盖模块

- `finance-system` - 财务系统（充值管理、提现审批、资金账目）
- `payment-channel` - 支付渠道（渠道配置、通道切换）
- `recharge-management` - 充值管理（充值记录、手动充值、自动充值）
- `withdrawal-management` - 提现管理（提现申请、审核、打款）

## 2. 常见业务对象

### 2.1 充值管理
- 充值订单：订单号、金额、渠道、状态、创建时间
- 充值方式：银行卡、USDT、Pix、信用卡、电子钱包
- 充值状态：待支付、支付中、成功、失败、关闭、超时
- 充值类型：自动充值（系统对接）、手动充值（人工确认）
- 充值优惠：首充优惠、充值返利、优惠券抵扣
- 充值限制：最小金额、最大金额、每日次数、风控限额

### 2.2 提现管理
- 提现申请：申请 ID、金额、银行卡/账户、手续费、状态
- 审核流程：初审、复审、终审、自动审核、人工审核
- 提现状态：待审核、审核通过、审核拒绝、打款中、已完成、已取消
- 提现限制：最小金额、最大金额、每日次数、余额要求
- 手续费规则：固定手续费、比例手续费、阶梯费率、免费额度
- 风控规则：异常提现、频繁提现、大额审核、黑名单

### 2.3 支付渠道
- 渠道配置：渠道名称、类型、费率、限额、状态
- 通道管理：主通道、备用通道、自动切换、权重配置
- 渠道对接：API 对接、回调配置、签名验签、对账
- 渠道监控：成功率、响应时间、可用性、告警
- 渠道对账：每日对账、差异处理、手动补单

### 2.4 资金账目
- 账户类型：主账户、优惠账户、任务账户、冻结账户
- 账目类型：充值、提现、投注、派奖、活动、补偿、转账
- 账目流水：流水号、类型、金额、余额、时间、备注
- 账户操作：存入、扣除、冻结、解冻、调账、划转
- 财务汇总：日汇总、周汇总、月汇总、按类型汇总

## 3. 高风险测试点

### 3.1 客户端

#### 充值页面
- 充值金额输入（档位选择/自定义输入）
- 充值方式选择（银行卡/USDT/Pix/其他）
- 优惠信息展示（首充优惠/返利活动）
- 支付二维码/支付链接生成
- 充值记录查询、状态展示
- 充值引导文案、支付教程

#### 提现页面
- 提现金额输入、可用余额展示
- 收款账户选择/新增（银行卡/USDT 账户）
- 手续费计算展示、实际到账金额
- 提现记录查询、状态跟踪
- 审核进度展示、拒绝原因
- 提现规则说明、限额提示

### 3.2 账服

#### 充值处理
- 充值订单创建、金额校验、频次限制
- 支付回调处理、签名验签、幂等性
- 充值成功入账、金币发放、优惠发放
- 充值超时关闭、订单状态同步
- 手动充值确认、审核流程
- 充值优惠计算、返利发放
- 风控检测：异常金额、频繁充值、黑名单

#### 提现处理
- 提现申请提交、余额校验、限额校验
- 手续费计算、实际到账金额
- 审核流程：自动审核规则、人工审核分配
- 审核通过/拒绝、打款处理
- 提现失败处理：退回金币、重试机制
- 风控检测：异常提现、频繁提现、大额预警
- 批量打款处理、渠道对接

#### 支付渠道管理
- 渠道配置增删改查、状态切换
- 通道权重配置、自动切换逻辑
- 渠道回调处理、订单状态更新
- 渠道对账：数据拉取、差异比对、补单处理
- 渠道监控：成功率统计、告警触发

#### 资金账目管理
- 账户余额查询、多账户管理
- 账目流水写入、事务一致性
- 账户操作：冻结/解冻/调账/划转
- 财务汇总统计、报表生成
- 异常账目处理、手动调账审批
- 账目审计：操作留痕、可追溯

## 5. 集成点

- **player-management**：玩家账户余额、玩家分层限额
- **marketing-activities**：充值活动联动、返利发放
- **risk-management**：风控规则触发、异常检测
- **operation-log**：操作留痕、审核轨迹
- **report-system**：财务报表、充值提现统计
- **notification-center**：到账通知、审核结果通知

## 6. 风险等级

| 风险等级 | 场景 | 说明 |
|----------|------|------|
| P0 | 充值入账错误 | 金额错误/重复入账，直接资金损失 |
| P0 | 提现重复打款 | 同一申请多次打款，资金损失 |
| P0 | 余额校验漏洞 | 超额提现、负余额 |
| P1 | 手续费计算错误 | 多收/少收手续费 |
| P1 | 审核流程绕过 | 未审核通过直接打款 |
| P1 | 渠道回调伪造 | 签名验签漏洞，虚假充值 |
| P2 | 状态不同步 | 前端展示与后端状态不一致 |
| P2 | 并发问题 | 同时提现的余额并发校验 |

## 7. 合规要求

- **反洗钱**：大额提现上报、频繁交易监控
- **实名认证**：提现前必须完成 KYC 认证
- **资金安全**：调账操作必须双人复核、留痕
- **数据合规**：支付信息加密存储、敏感信息脱敏
