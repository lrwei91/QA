# 运营活动领域

## 1. 覆盖模块

- 任务系统（日常任务、成长任务、活动任务）
- 锦标赛（FG/YELLOWBAT 等厂商锦标赛）
- 充值活动（充值返利、充值抽奖、首充活动）
- 洗码活动（洗码返利、流水返利、亏损返还）
- 营销活动（领券中心、签到、转盘、幸运转盘、邀新有礼）
- 厂商活动记录（FC 抽红包、FG 免费旋转等）
- 客户后台（客服工单、客户管理）
- 报表系统（数据报表、统计报表、分析报表）
- 活动运营（活动配置、活动推广）
- 前台显示（前端展示优化、UI 展示）

## 2. 常见业务对象

### 2.1 任务系统
- 任务类型：日常任务、成长任务、一次性任务、限时任务
- 任务条件：投注任务、充值任务、连赢任务、连输任务
- 任务奖励：金币、道具、优惠券、抽奖机会
- 任务状态：进行中、已完成、已领取、已过期
- 任务刷新：每日刷新、每周刷新、手动刷新

### 2.2 锦标赛
- 锦标赛类型：按游戏厂家（FG/TADA/YELLOWBAT/FC）、按时间（日/周/月）
- 报名方式：免费报名、付费报名、自动报名、邀请制
- 排名规则：按赢利、按投注额、按连赢、按特定条件
- 奖励池：总奖金、名次分配、奖励类型（金币/道具/实物）
- 结算方式：自动结算、手动结算、定时结算
- 状态流转：未开始→进行中→已结束→已结算

### 2.3 充值活动
- 活动类型：首充活动、每日充值、累计充值、阶梯返利、复充返利
- 充值档位：单笔满额、累计满额、多档位选择
- 返利类型：即时返利、手动领取、定时发放
- 返利形式：金币、优惠券、抽奖机会、道具
- 限制条件：每日次数限制、总参与次数、玩家资格限制

### 2.3.1 复充返利活动
- 活动定义：玩家在完成首充后，再次充值时触发返利
- 触发条件：
  - 必须已完成首充
  - 首充后账户余额 ≤ 配置的阈值（如 R$100）
  - 首充金额 ≥ 配置的最小值（如 R$100）
- 返利计算：
  - 返利基数：首充金额（可配置）或 复充金额
  - 返利比例：后台配置（如 100%）
  - 返利金额 = 基数 × 返利比例
  - 赠送金额上限：后台配置最高返利额度
- 触发时机：
  - 玩家首充后，首次余额 ≤ 配置值时
  - 或 复充金额达到配置档位时
- 弹框机制：
  - 检测时机：登录/大刷、退出游戏回大厅、后台定时器主动推送
  - 弹框频率：按配置（如每 10 小时弹 1 次）
  - 限制：玩游戏时不允许弹框
- 限制条件：
  - 每个玩家最多参加 1 次
  - 相同 CPF 限购次数（防刷）
  - 相同 IP 限购次数（防刷）
- 叠加规则：可与任意其他充值活动叠加
- 活动记录：
  - 用户 ID、复充金额、是否完成、首充金额、返利金额、完成时间

### 2.4 洗码活动
- 洗码定义：有效投注、有效输赢、流水计算
- 洗码比例：不同游戏类型不同比例（电子/真人/体育）
- 结算周期：每日结算、每周结算、每月结算
- 返利形式：金币返利、优惠券、升级奖励
- 风控规则：异常投注检测、对刷检测、限制条件

### 2.5 营销活动
- 领券中心：优惠券领取、使用条件、有效期
- 签到系统：连续签到、累计签到、补签机制
- 幸运转盘：抽奖次数、奖品池、保底机制
- 礼包系统：新手礼包、等级礼包、节日礼包
- 兑换商店：金币兑换、道具兑换、实物兑换

### 2.6 客户后台
- 客户列表：客户信息展示、筛选与排序
- 客户详情：基本信息、游戏记录、资金流水
- 客服工单：工单创建、处理、分配、关闭
- 客户备注：添加备注、标签分类、历史备注查看
- 客户筛选：按注册日期、投注额、输赢、标签筛选
- 权限控制：不同客服角色可见范围、数据范围限制

### 2.8 免费旋转 (Free Spin)
- 获取方式：活动赠送、任务奖励、充值赠送、客服派发、系统补偿
- 旋转模式：直通模式（直接旋转）、手动模式（逐次点击）、自动模式（批量旋转）
- 使用规则：指定游戏使用、有效期限制、单次使用数量、中奖上限
- 下分逻辑：中奖金额转入主钱包、下分失败处理、批量补发机制
- 记录管理：
  - 获取记录：期数、活动名称、获取时间、获取数量、获取方式、状态
  - 使用记录：期数、游戏名称、使用时间、使用数量、中奖金额、状态
- 统计报表：获取统计、使用统计、中奖统计、ROI 分析
- 限制条件：每日获取上限、总持有上限、使用游戏限制、有效期校验

### 2.7 报表系统
- 报表类型：玩家报表、游戏报表、活动报表、资金报表
- 统计维度：时间、游戏类型、玩家等级、渠道来源
- 数据展示：表格、趋势图、柱状图、饼图
- 筛选条件：日期范围、游戏厂家、玩家分组、状态筛选
- 数据导出：Excel 导出、CSV 导出、定时导出
- 报表刷新：手动刷新、定时刷新、缓存策略

### 3.2 账服

#### 任务系统
- 任务条件判定逻辑（投注额/连赢/连输/充值）
- 任务进度更新、状态流转
- 奖励发放、金币入账、道具发放
- 任务刷新逻辑、过期清理
- 多任务并行处理、并发奖励发放

#### 锦标赛
- 报名逻辑校验（资格/余额/次数）

- 定时结算、奖励批量发放
- 厂商数据回调处理
- 并列排名处理、破规则

#### 充值活动
- 充值金额校验、档位匹配
- 返利资格判定（首充/每日/累计）
- 返利发放、次数限制校验
- 活动配置（时间/档位/奖励）

#### 洗码活动
- 洗码计算逻辑（不同游戏类型比例）
- 有效投注判定（排除对刷/异常）

#### 免费旋转
- 免费旋转获取/使用记录查询接口
- 获取记录筛选条件解析（时间范围、活动名称、状态）
- 使用记录筛选条件解析（时间范围、游戏名称、状态）
- 分页逻辑、排序规则（按时间倒序）
- 记录数据一致性（获取数量=使用数量 + 剩余数量）
- 有效期校验、过期清理
- 下分接口幂等性、失败重试、批量补发

## 5. 集成点

- **财务系统**：充值活动依赖充值记录、洗码活动依赖金币账目
- **玩家管理**：玩家资格判定、玩家分层参与条件
- **游戏管理**：游戏数据同步、投注记录获取
- **报表系统**：活动数据报表、参与统计
- **厂商活动记录**：厂商活动数据对接
- **客户后台**：客服工单与客户数据联动

## 6. 风险等级

| 风险等级 | 场景 | 说明 |
|----------|------|------|
| P0 | 奖励发放错误 | 多发/少发/错发，直接资金损失 |
| P0 | 排名计算错误 | 锦标赛排名错误，影响公平性 |
| P1 | 资格判定错误 | 不符合条件的玩家参与获利 |
| P1 | 数据不同步 | 前端进度与后端数据不一致 |
| P2 | 文案错误 | 活动规则展示错误，用户投诉风险 |
| P2 | 并发问题 | 多人同时领取奖励的幂等性 |
