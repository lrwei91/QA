# QA 测试用例生成知识库研究方向

## 核心目标

将需求文档（PRD / 页面 / 接口 / 描述）转换为结构化测试用例，并支持：
- Excel 测试用例生成与追加
- 多语言 JSON 校验与导出
- 测试用例索引与复用

## 关键问题

1. **如何准确识别模块归属**
   - 基于关键词匹配模块索引
   - 支持用户确认和自定义输入
   - 保持命名一致性

2. **如何覆盖高风险测试场景**
   - 优先覆盖 P0/P1 级风险
   - 关注资金安全、数据一致性
   - 防刷机制验证

3. **如何保持测试用例的可执行性**
   - 操作步骤明确具体
   - 预期结果可验证
   - 不出现空话套话

4. **如何复用历史测试用例**
   - 建立测试用例索引
   - 支持增量补充
   - 保持用例结构稳定

## 研究领域

### 核心领域

- [[运营活动]] - 任务系统、锦标赛、充值活动、洗码活动、营销活动
- [[财务系统]] - 充值提现、支付渠道、账目管理、资金安全
- [[账号访问]] - 认证、账号、权限、组织、KYC
- [[代理管理]] - 代理系统、佣金、团队管理、ROI

### 工程规范

- 测试用例存储约定
- 更新治理规则
- 领域文档模板

### 测试设计

- 测试覆盖分类
- 输出模板规范
- 平台划分规则

## 研究范围

**包含**：
- 从需求文档生成结构化测试用例
- 测试用例的增量补充和更新
- 多语言文案校验
- 测试用例索引管理

**不包含**：
- UI 自动化执行
- Playwright 脚本运行
- 测试结果验证

---

## 使用说明

本知识库与 QA 项目的 `engine/references/` 目录协同工作：
- 领域知识存储在 `knowledge/wiki/entities/`（中文命名）
- 核心规则和模板存储在 `engine/references/`
- `testcase-generate` skill 优先读取新知识库，失败时回退到旧结构
